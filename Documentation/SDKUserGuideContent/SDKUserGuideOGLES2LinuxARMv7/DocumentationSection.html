Documentation

The following documentation is available in the Documentation folder in the SDK package:

  • PowerVR Series 5 Architecture Guide for Developers

    The purpose of this document is to provide graphics programmers with an overview of the PowerVR Series 5 (SGX & SGX-MP) graphics hardware architecture, while also highlighting why some performance recommendations make such a significant difference to graphics rendering on PowerVR platforms. Furthermore, the information in this document outlines the purpose of each hardware module for which PVRTune provides performance counters.

    This document is available here.

  • PowerVR Performance Recommendations

    Performance guidelines and best practices for writing applications for PowerVR hardware.

    This document is available here.

  • PFX Language Format Specification

    Describes PowerVR's shader effect file format: PFX.

    This document is available here.

  • PVR File Format Specification

    Describes PowerVR's texture container format: PVR.

    This document is available here.

  • POD File Format Specification

    Describes PowerVR's 3D data container format: POD.

    This document is available here.

  • Parallax Bumpmapping Whitepaper

    Parallax bumpmapping is a technique that adds greater detail to models without adding further geometry. This whitepaper describes the technique, the implementation available in our Examples, and the optimization steps taken by PowerVR Developer Technology to ensure its efficiency with PowerVR hardware.

    This document is available here.

  • 3D Navigation Rendering Techniques Whitepaper

    Describes techniques for preparing and rendering real-world navigation data efficiently in 3D. Companion to Navigation3D available in the ExamplesSection.html#AdvancedSection. For "2.5D" techniques please see the "PowerVR Navigation Rendering Techniques" document

    This document is available here.

  • Edge Detection Whitepaper

    Discusses edge detection in 3D rendering as implemented in Edge Detection in our Examples.

    This document is available here.

  • Navigation Rendering Techniques Whitepaper

    Describes preparing and rendering real-world navigation data for rendering using PowerVR hardware acceleration. A "2.5D" approach is explored (for 3D rendering please see the "PowerVR 3D Navigation Rendering Techniques" document). This is illustrated in Navigation in our ExamplesSection.html#AdvancedSection.

    This document is available here.

  • Navigation Rendering Techniques Whitepaper

    Describes preparing and rendering real-world navigation data for rendering using PowerVR hardware acceleration. A "2.5D" approach is explored (for 3D rendering please see the "PowerVR 3D Navigation Rendering Techniques" document). This is illustrated in Navigation in our ExamplesSection.html#AdvancedSection.

    This document is available here.

  • Post Processing Effects Whitepaper

    This document describes a general approach to post processing and provides an example of a real-time, bloom effect that can be implemented for mobile platforms utilizing PowerVR SGX hardware. The example project is one of our Advanced examples.

    This document is available here.

  • Shader Based Water Effects Whitepaper

    This white paper deals with the implementation of realistic water effects on PowerVR hardware and is a companion to the Water demo to be found with our other ExamplesSection.html#AdvancedSection.

    This document is available here.

  • UI Example Whitepaper

    The UI Example in ExamplesSection.html#AdvancedSection shows some ways to implement user interfaces efficiently using PowerVR hardware acceleration. This white paper discusses these techniques and other optimizations possible for user interfaces.

    This document is available here.
